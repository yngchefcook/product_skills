# Примеры: Wireframe Spec

## Пайплайн-режим (вход `story_map` → выход `wireframes`)

### Вход (сокращённо)
`story_map` Финоко: Backbone Знакомство → Ввод данных → Расчёт и объяснение → Работа с
моделью → Экспорт. MVP-эпики: диалоговый ввод (US-002), расчёт с объяснением (US-003).

### Выход (`wireframes`)

```markdown
# Wireframes: Финоко · Платформа: Desktop (B2B-инструмент)

## Карта экранов
╭──────────────╮    ╭──────────────╮    ╭──────────────╮
│  Диалог-ввод │──▶│  Дашборд      │──▶│  Экспорт      │
│  данных      │    │  юнит-эконом. │    │  для инвест.  │
╰──────────────╯    ╰──────────────╯    ╰──────────────╯

## Экран: Диалог ввода данных
**Контекст:** фаундер начал сбор модели · **Цель:** ввести цифры без формул · **Покрывает:** US-002, SS-001

### Wireframe
╭──────────────────────────────────────────────────────────────╮
│  ⌂ Финоко                                            ◯        │ ← 1
├──────────────────────────────────────────────────────────────┤
│   Шаг 3 из 8   ████████░░░░░░░░░░░░                            │ ← 2
│                                                                │
│   Сколько в среднем стоит привлечь одного платящего клиента?  │ ← 3
│   ℹ Это CAC — сумма маркетинга и продаж, делённая на число    │
│     новых клиентов за период.                                 │
│                                                                │
│   ╭────────────────────────────╮                              │
│   │ 4500                      ₽ │                             │ ← 4
│   ╰────────────────────────────╯                              │
│                                                                │
│   [ Не знаю — помоги прикинуть ]                              │ ← 5
│                                                                │
│                          ╭────────────╮  ╭────────────────╮   │
│                          ┆   Назад     ┆  │   Далее         │  │ ← 6
│                          ╰────────────╯  ╰────────────────╯   │
╰──────────────────────────────────────────────────────────────╯

### Аннотации
| # | Элемент | Поведение |
|---|---|---|
| 1 | Хедер | Логотип · ◯ профиль (вход опционален) |
| 2 | Прогресс | Показывает, сколько вопросов осталось — снижает тревогу |
| 3 | Вопрос + ℹ | Один показатель за раз, термин поясняется на месте (US-002) |
| 4 | Поле ввода | Валидация числа, единица измерения справа |
| 5 | «Не знаю» | Ассистент предлагает способ прикинуть (SS-001, дефолт-допущение) |
| 6 | Назад/Далее | Далее активна при заполненном/пропущенном поле |

### Состояния
- Empty: поле пустое, «Далее» ведёт к «Не знаю»-ветке.
- Error: нечисловой ввод → ⚠ «Введите число».
- Loading: между шагами — короткий индикатор пересчёта.

### Переходы
- Далее → следующий вопрос / по завершении → Дашборд.
- «Не знаю — помоги прикинуть» → подсказка с вариантами оценки.

## Экран: Дашборд юнит-экономики
**Контекст:** данные собраны · **Цель:** понять экономику с объяснением · **Покрывает:** US-003

### Wireframe
╭──────────────────────────────────────────────────────────────╮
│  ⌂ Финоко                                            ◯        │
├──────────────────────────────────────────────────────────────┤
│   Юнит-экономика · черновик                    [ Экспорт ↗ ]  │ ← 1
│                                                                │
│   ╭───────────────╮ ╭───────────────╮ ╭───────────────╮       │
│   │ CAC           │ │ LTV           │ │ LTV/CAC       │       │ ← 2
│   │ 4 500 ₽       │ │ 12 000 ₽      │ │ 2.7  ✓        │       │
│   │ ℹ как считали │ │ ℹ как считали │ │ норма > 3     │       │
│   ╰───────────────╯ ╰───────────────╯ ╰───────────────╯       │
│                                                                │
│   ⚠ Маржа на клиента ниже целевой — посмотрите, где теряете.   │ ← 3
│                                                                │
│   ────────────────────────────────────────────────────────    │
│   Что это значит                                               │ ← 4
│   На каждый вложенный рубль вы возвращаете 2.7 — неплохо для    │
│   ранней стадии, но до здоровой нормы (>3) есть куда расти.    │
╰──────────────────────────────────────────────────────────────╯

### Аннотации
| # | Элемент | Поведение |
|---|---|---|
| 1 | Экспорт | → экран экспорта для инвестора (US-005) |
| 2 | Карточки метрик | Значение + ℹ раскрывает метод расчёта (прозрачность = доверие) |
| 3 | Алерт | Подсветка проблемного места (US-003, AC) |
| 4 | «Что это значит» | Объяснение выводов простым языком |

### Состояния
- Empty (данных мало): карточки показывают «нужно больше данных» + ссылка добить ввод.
- Error (LTV<CAC): красный акцент + объяснение риска.

### Переходы
- Экспорт → Экспорт для инвестора · клик по ℹ → раскрытие расчёта · «изменить данные» → Диалог-ввод.
```
