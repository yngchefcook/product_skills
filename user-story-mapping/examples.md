# Примеры: User Story Mapping

## Пайплайн-режим (вход `brief` + `personas` → выход `story_map`)

### Вход (сокращённо)
- `brief`: Финоко — собрать юнит-экономику за 15 минут без финансиста; критерий — объяснение
  каждого показателя, обновляемость модели, экспорт для инвестора.
- `personas`: Артём (фаундер-нефинансист), Дана (early adopter).

### Выход (`story_map`)

```markdown
# User Story Map: Финоко

## Backbone (путь пользователя)
Знакомство → Ввод данных → Расчёт и объяснение → Работа с моделью → Экспорт/шеринг

## Release 1.0 (MVP — walking skeleton)

### [EPIC-001] Знакомство и старт
---
**[US-001][Фаундер] Как фаундер, я хочу понять за минуту, что умеет продукт, чтобы решить, стоит ли вводить данные.**
- [ ] На старте — короткий пример «что получится на выходе»
- [ ] Можно начать без регистрации
- Notes: снижаем трение входа (боль «нет времени»).

### [EPIC-002] Ввод данных через диалог
---
**[US-002][Фаундер] Как фаундер, я хочу вводить цифры в диалоге простыми словами, чтобы не разбираться в формулах.**
- [ ] Ассистент спрашивает по одному показателю за раз
- [ ] Каждый термин поясняется в момент запроса
- [ ] Можно ответить «не знаю» → ассистент предлагает как прикинуть
- Notes: [⚠️ Risk] слишком длинный опрос убьёт активацию — держать коротким.
---
**[SS-001][Система] Как система, я хочу подставлять разумные значения по умолчанию, чтобы фаундер не застревал на незнакомых полях.**
- [ ] Дефолты помечены как допущения и легко меняются

### [EPIC-003] Расчёт и объяснение
---
**[US-003][Фаундер] Как фаундер, я хочу видеть CAC/LTV/маржу/окупаемость с объяснением, чтобы понимать смысл, а не только число.**
- [ ] Каждый показатель: значение + «что это и почему важно» + как посчитан
- [ ] Подсветка проблемных мест (например, LTV < CAC)
- Notes: прозрачность расчёта = доверие (главное рискованное допущение).

### [EPIC-004] Работа с моделью
---
**[US-004][Фаундер] Как фаундер, я хочу менять вводные и сразу видеть, как поменялись выводы, чтобы проигрывать сценарии.**
- [ ] Пересчёт при изменении любого показателя
- [ ] Видно, что именно изменилось

### [EPIC-005] Экспорт
---
**[US-005][Фаундер] Как фаундер, я хочу выгрузить сводку для инвестора, чтобы ответить на вопрос об экономике.**
- [ ] Экспорт в презентабельный формат

## Release 1.1
### [EPIC-003] (расширение)
**[US-010][Фаундер] Как фаундер, я хочу сравнить свою экономику с бенчмарками, чтобы понять, где я отклоняюсь.** [assumption] нужен источник бенчмарков.

## Gap Analysis & Risks
- **Пропущенные функции:** сохранение нескольких проектов/моделей; напоминания обновить.
- **Открытые вопросы:** глубина модели, достаточная инвестору; нужны ли шаблоны по отраслям.
- **Риски:** [⚠️ Risk] доверие к AI-цифрам; [⚠️ Risk] трение ввода данных снижает retention.
```
